Data structures (internal, 4 separate Maps):
  _strings: Map<key, { value: string, expiresAt: number|null }>
  _sets:    Map<key, Set<string>>
  _hashes:  Map<key, Map<field, string>>
  _sorted:  Map<key, Array<{score: number, member: string}>>   // sorted ascending
  _lists:   Map<key, Array<string>>

TTL strategy: lazy expiry — check expiresAt on every read, return null/0 if expired.
No background janitor needed.

Snapshot persistence:
  SNAPSHOT_PATH = process.env.STORE_SNAPSHOT || '/var/lib/linbo-native/store.json'
  flushToDisk() — atomicWrite (use existing src/lib/atomic-write.js)
  loadFromDisk() — called once at module load (startup recovery)
  On load: ALWAYS clear these keys: sync:isRunning, linbo:update:lock, imgsync:lock, imgpush:lock
  On load: NEVER restore host:status:* keys (volatile)
  On load: NEVER restore ops:op:* keys (volatile)
  Skip expired entries during restore

Snapshot exclusion filter (do NOT serialize these key patterns):
  /^host:status:/    — volatile host status
  /^ops:/            — volatile operation queue
  /^sync:isRunning$/ — sync lock (clear on startup)
  /^linbo:update:lock$/ — update lock
  /^imgsync:lock$/   — image sync lock
  /^imgpush:lock$/   — image push lock
  /^rl:/             — rate limit counters

Exports required:
  module.exports = { client, reset, flushToDisk, loadFromDisk }

  client — object with all methods below
  reset() — synchronous, clears ALL 5 Maps (for test isolation only)
  flushToDisk(path?) — async, writes snapshot excluding volatile keys
  loadFromDisk(path?) — async, restores snapshot, clears lock keys

Key method contracts (verified in research against all 17 importer call sites):

  client.status = 'ready'   // MUST be synchronous property at module load time
                             // host-status.worker.js line 61 checks: if (client.status !== 'ready') return;

  get(key) → string|null
    - check expiry, return null if expired or missing
    - return string value (callers do their own JSON.parse if needed)

  set(key, value, ...opts)
    - opts=[] → plain set, no TTL
    - opts=['NX','EX', ttl] → conditional set with TTL
      returns 'OK' if key did NOT exist (set successfully)
      returns null if key existed (NX = not exists condition failed)
    NOTE: linbo-update.service.js calls: redis.set(LOCK_KEY, runId, 'NX', 'EX', LOCK_TTL)
    This uses the redis.js wrapper set(), NOT client.set() — so client.set() variadic is
    still needed because other code may call client.set() directly

  setex(key, ttl, value)
    - set with TTL in seconds

  del(...keys)
    - variadic: del('key1') or del('key1','key2') or del(...arrayOfKeys) via spread

  expire(key, ttl)
    - set expiresAt on existing key
    - if key does not exist, no-op (return 0)
    - returns 1 if TTL was set

  exists(key) → 1|0
    - 1 if key present and not expired
    - 0 otherwise

  mget(...keys) → Array<string|null>
    - ioredis accepts mget(k1, k2, k3) or mget([k1, k2, k3])
    - handle both flat args and single array arg

  sadd(key, ...members), smembers(key), sismember(key, member) → 1|0
  srem(key, ...members), scard(key) → number

  hmset(key, obj)
    - obj is a plain JS object: { field1: val1, field2: val2 }
  hset(key, field, value)
  hget(key, field) → string|null
  hgetall(key) → object|null
    - returns null if key missing (NOT empty object)
    - returns plain JS object { field: value, ... }

  zadd(key, score, member)
    - score is a number (timestamp in sync-operations)
  zrevrange(key, start, stop) → string[]
    - stop=-1 means all (ioredis convention)
  zrem(key, ...members)
  zremrangebyrank(key, start, stop)
    - start=0, stop=-N means keep last N entries

  rpush(key, ...values)   — append to list (right push)
  lrange(key, start, stop) → string[]
    - stop=-1 means all
  lrem(key, count, value)
    - count=0 removes all occurrences (ioredis convention used in image-sync.service.js)
  lpop(key) → string|null   — for future use, implement now
  llen(key) → number        — for future use, implement now

  pipeline()
    - returns a chainable pipe object
    - each method call on pipe queues [method, args]
    - pipe.exec() executes all queued commands sequentially
    - CRITICAL: exec() returns Promise<Array<[null, result]>> — ioredis tuple format
    - sync.service.js: results.filter(([err, val]) => !err && val).map(([, val]) => JSON.parse(val))
    - Also supports: client.pipeline(arrayOfCommands) where arrayOfCommands = [['del', key], ...]
      used in redis.js delPattern: client.pipeline(keys.map(k => ['del', k])).exec()

  scanStream({match='*', count=100})
    - returns EventEmitter with .pause() .resume() .destroy() methods
    - on next tick: emits 'data' with all matching keys, then emits 'end'
    - match uses glob pattern: '*' matches all, 'foo:*' matches keys starting with foo:
    - scan ALL Maps: strings + sets + hashes + sortedSets + lists

  publish(channel, msg) → no-op async function
  subscribe(channel, cb) → no-op async function
  call(command, ...args) → throws Error('store.js: call() not implemented')
