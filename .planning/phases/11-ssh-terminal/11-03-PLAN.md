- Detect /root/.ssh/id_rsa is a directory → remove it → generate fresh 3072-bit RSA key

- Back up the existing .env and write a new one with LINBO_CLIENT_SSH_KEY=

IMPORTANT: After setup.sh runs, there will be a new .env with regenerated secrets (JWT_SECRET, INTERNAL_API_KEY, RSYNC_PASSWORD). The admin password resets to "Muster!". This is expected.

  <action>
The freshly generated /root/.ssh/id_rsa.pub must be injected into linbofs64's .ssh/authorized_keys so live clients accept the key. Run update-linbofs on the server:

```bash
ssh root@10.40.0.10 '
  echo "=== Rebuilding linbofs64 (injects id_rsa.pub into authorized_keys) ==="
  /usr/sbin/update-linbofs
  echo "=== linbofs64 timestamp ==="
  ls -la /srv/linbo/linbofs64
  echo "=== Restarting linbo-api ==="
  systemctl restart linbo-api
  sleep 3
  systemctl status linbo-api --no-pager | head -20
  echo "=== API logs (last 20 lines) ==="
  journalctl -u linbo-api -n 20 --no-pager
'
```

update-linbofs takes approximately 2 minutes. Wait for it to complete.

After restart, verify the API loads the key successfully by checking logs for:
- `[SSH] Loaded LINBO client key from /etc/linuxmuster/linbo/linbo_client_key`  (appears on first SSH use)
- OR absence of `SSH private key not available` errors in the startup log

If a client is currently booted from the OLD linbofs64 (without the new public key), it will still reject SSH connections. The client must reboot from LINBO PXE to pick up the new linbofs64. If no client is available for live testing, SSH-01 is verified by key file existence; SSH-02 and SSH-03 require the checkpoint below.
